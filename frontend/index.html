DB command template syntax

Configure per operation under DB command templates. See sql/README.md in the repository for full details.

Execution modes

  • Function call — Write the call only (no SELECT). Placeholders become $1, $2, … For remove_role, ${rolename} is an alias for ${loginname}.
  • SQL statement — Single statement, no SELECT. Role names are embedded as identifiers (not quoted). Examples: DROP ROLE ${loginname}; GRANT ${parent_roles} TO ${loginname}.
  • PL/pgSQL block — Inner statements only; the app wraps DO $dbaccounts$ BEGIN … END. Semicolons allowed inside.

Placeholders

Use ${field} in templates. Common fields: ${loginname}, ${fullname}, ${email}, ${parent_role}, ${parent_roles}, ${new_password}, ${rolename} (remove_role alias).

Function mode: do not use raw $1 or SELECT in the template. Statement/block mode: role names are identifiers, not bind parameters.
